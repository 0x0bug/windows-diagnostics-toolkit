Troubleshooting case

Windows Time service is stopped

Inspect time-service context without starting the service, changing peers, or requesting resynchronization.

pwsh -NoProfile -File .\Invoke-WindowsDiagnostics.ps1 -Time -PrivacyMode -ExportMarkdown

What it checks

W32Time state and start mode, domain membership, timezone, local and UTC clock, configured source, and verbose status.

Optional evidence

The standalone module can include recent Time-Service warning and error events.

Interpretation

A stopped service can be normal on a non-domain computer until Windows needs it. Use the source, status, domain context, and events together rather than treating service state alone as a failure.
