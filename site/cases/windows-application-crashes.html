Troubleshooting case

Windows application keeps crashing

Confirm repeated failures using Event Log metadata and recent dump-file metadata without opening dump contents or changing Windows Error Reporting.

pwsh -NoProfile -File .\Invoke-WindowsDiagnostics.ps1 -Crashes -PrivacyMode -ExportMarkdown

What it checks

Application Error, Windows Error Reporting, Application Hang, system BugCheck events, and recent dump-file metadata.

What it does not do

It does not debug the dump, delete files, restart applications, or change recovery settings.

Use the result

Look for repeated providers, timestamps, application names, and fresh dumps. A repeated pattern is useful evidence for a vendor issue, driver conflict, or application-specific investigation.
