Troubleshooting case

Unexpected default route or proxy

Compare active adapters, gateways, route metrics, DNS, DHCP, and proxy configuration without changing the network stack.

pwsh -NoProfile -File .\Invoke-WindowsDiagnostics.ps1 -Network -PrivacyMode -ExportMarkdown

Routing context

Default routes appear first with next hop, interface, and effective metric so competing paths are easier to identify.

Proxy context

WinINET and WinHTTP settings are reported read-only. Credentials and sensitive URL query values are removed.

Interpretation

The lowest effective default-route metric usually identifies the preferred path. Virtual switches and tunnel adapters may legitimately add routes, so compare the result with the software currently running.
