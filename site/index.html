One report, ten diagnostic areas

Useful context before guesswork

The report starts with an aggregated findings summary and keeps the detailed raw diagnostic context below it.

System and performance

Windows build, CPU, memory, GPU, uptime, pagefile, CPU snapshot, and top processes.

Security posture

Microsoft Defender, Windows Firewall, Secure Boot, TPM, and BitLocker status.

Network context

Adapters, DHCP, DNS, gateways, routes, proxy settings, and basic reachability checks.

Crashes and Event Logs

Application failures, hangs, BugCheck events, dump metadata, and recent critical errors.

Disk and services

Physical disk health, free space, automatic services not running, startup, and scheduled tasks.

Time and updates

W32Time status, timezone, configured source, reboot indicators, recent updates, and update services.

Designed for support reports

Local first. Share only what you choose.

  • Read-only:The production scripts collect diagnostic state and do not apply fixes or configuration changes.
  • No telemetry:Reports stay in the output directory selected by the user. Nothing is uploaded.
  • Privacy Mode:Computer names, users, paths, addresses, MACs, SIDs, GUIDs, and device identifiers are replaced with typed tokens in combined reports.
  • Auditable:The source, validation policy, and dependency-free tests are public on GitHub.
Before posting a report publicly: run the combined wrapper with -PrivacyMode and still review the final file manually.
