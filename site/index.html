A read-only Windows diagnostics tool for PowerShell

Generate a local support report covering security, performance, networking, crashes, disks, services, Event Logs, time synchronization and Windows Update. A typical full run completes in about 30 seconds.

Run the diagnostics View an example report Support report guide
PowerShell
git clone https://github.com/0x0bug/windows-diagnostics-toolkit.git
cd windows-diagnostics-toolkit
pwsh -NoProfile -File .\Invoke-WindowsDiagnostics.ps1 `
  -All -PrivacyMode -ExportMarkdown

Designed for support reports

Local first. Share only what you choose.

  • Read-only:The production scripts collect diagnostic state and do not apply fixes or configuration changes.
  • No telemetry:Reports stay in the output directory selected by the user. Nothing is uploaded.
  • Privacy Mode:Common identifying values are replaced with stable typed tokens in combined reports.
  • Auditable:The source, validation policy and dependency-free tests are public on GitHub.
Before posting a report publicly: run the combined wrapper with -PrivacyMode and still review the final file manually.

Common questions

Windows Diagnostics Toolkit FAQ

Does the toolkit change Windows settings?

No. The production scripts are designed to collect and report state without applying automatic fixes, changing services, editing the registry or reconfiguring the network.

Does it upload diagnostic data?

No. Reports remain local. The project has no telemetry, remote collection or automatic upload feature.

Does it require administrator rights?

Most checks work without elevation. Some Windows sources may return less detail when the current user does not have permission to read them.

What should I run before sharing a report?

Use -All -PrivacyMode -ExportMarkdown, then inspect the final file and remove any remaining information you do not want to publish.
